# Шпаргалка по Git

## 1. Что такое Git

> Git - это машина времени для кода. 

Он запоминает все, что когда либо делали на проекте: файлы, строки, даже случайные опечатки и исправления.

## 2. Для чего вообще нужен Git

> Git - это страховка от катострофы. 

Он помагает: сохранять историю, работать в команде без хаоса, делится кодос с миром.

## 3. Как работает Git

> Git хранит не только файлы — он хранит моменты. 

Каждый коммит - это снимок состояния проекта, а ветки - паралельные линии реальности, где можно творить без страха. Если после апдейта все разволилось - просто откатываешь на прошлый коммит и делаешь вид что ничего не было (но не забывай история останется навсегда).

## 4. Лог

> Лог - это дневник который не умеет врать. 

Каждая строчка - кусочек истории проекта: кто, когда и что сделал. Можно проследить путь от первого коммита до все сново работает.

## 5. Установка Git

> Чтобы управлять кодом, сначало нужно приручить Git. 

#### Установка на MacOS

Сначало проверь, вдруг Git уже есть.
Открой терминал и напиши:

    git --version

Если  видишь что- то вроде:

    git version 2.xx.x (Apple Git-xxx)

\- поздравляю у тебя уже есть системная версия Git (ставится вместе с Xcode Tools). 
Если выдало сообщение вроде

> “Command line tools not found…”

то просто соглашайся, и macOS сам предложит установить инструменты разработчика.

Если хочешь свежий, чистый Git как я, установи Homebrew:

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Проверь, что Homebrew установился:

    brew --version

Далее, установи Git.

    brew install git

и проверь:

    git --version

Теперь версия должна быть новая ө что то вроде:

    git version 2.50.0


## 6. Настраиваем Git под себя

> Git должен знать кто ты, прежде чем ты начнешь творить чудеса

    git config --global user.name "Твое имя"
    git config --global user.email "Твая эл. почта"

Каждый коммит в репозитории подписывается твоим именем и эл.почтой - это как подпись для истории проекта.

Важно!!!

--global - значит для всех проектов на этом компьютере. Если его убрать настройка будет только для текущего репозитория.

Чтобы проверить как твое имя/почта записалось достаточно набрать команду в консоли:

    git config --list

## 7. После Git следующий зверь - это GitHub

> С Git ты говоришь один на один, GitHub - это уже социальная сеть для репозиториев

GitHub - это место, где твой локальный репозиторий получает подписку в облаке.
Если Git это твой дневник, GitHub делает из него публикацию для команды

## 8. Регистрация на GitHub

> Прежде чем пушить - нужно появится в списке живых

- Заходим на https://github.com/
- Нажимаем Sign up
- Вводим email — лучше рабочий,
- Вводим username — то, под чем ты будешь виден в коммитах,
- Придумываем пароль — да, надёжный, а не 123qwe.
- Подтверждаем почту (GitHub пришлёт письмо).
- Выбираем тариф

Все у тебя есть свой аккаунт

## 9. Создаем репозиторий на GitHub

> Каждому проекту нужен дом. GitHub — это твой сервер с табличкой "Добро пожаловать!"

- На главной странице нажми New Repository
- Придумай имя и выбери: public - видят все, private - только ты и избранные тобой люди ;)
- Нажми на create repository

На экране появится подсказка как подключить локальный Git

## 10. Настраиваем SSH-ключ для GitHub

> Чтобы GitHub узнавал тебя без пароля — нужен ключ. Как у замка, только цифровой.

Каждый раз при git push GitHub может требовать логин и пароль.
SSH-ключ решает это: он даёт твоему компьютеру удостоверение личности, чтобы GitHub знал — это точно ты.

Шаг 1. Проверяем, есть ли ключ

    ls -al ~/.ssh

Если там уже есть файлы вроде id_rsa или id_ed25519 — поздравляю, ключ уже есть.
Если нет — создаём новый.

Шаг 2. Создаем новый SSH-ключ

    ssh-keygen -t ed25519 -C "твоя.почта@example.com"

Нажимай Enter на всех вопросах — по умолчанию всё отлично.
Ключи появятся в папке ~/.ssh:

- id_ed25519 — секретный ключ (никому не показывай)
- id_ed25519.pub — открытый (его добавляем в GitHub)

Шаг 3. Добавляем ключ в GitHub

Скопируй открытый ключ и перейди:

Settings → SSH and GPG keys → New

Вставь туда скопированный ключ, задай имя и нажми на Add SSH key.

Шаг 4. Проверь соединение

ssh -T git@github.com

Если все ок, GitHub ответит:

    You've successfully authenticated, but GitHub does not provide shell access.

Готово — теперь ты можешь пушить, пуллить и клонировать без пароля.

## 10. Подключаем локальный репозитории к GitHub

> Теперь пора связать твою машину времени с облаком

Ты уже настроил Git и создал репозиторий на GitHub. Осталось их познакомить.

Шаг 1. Проверь что Git инициализирован.
Перейди в папку проекта и выполни:

    git status

Если Git не знает про эту папку, инициализируй его

    git unit

Шаг 2. Добавь файлы

    git add . (все файлы в папке)
    git commit -m "Первый коммит: начало проекта"

Шаг 3. Привяжи локальный репозиторий к GitHub

Скопируй ссылку из своего нового репозитория и выполни:

    git remote add origin https://github.com/твое_имя/имя_репозитория.git

Шаг 4. Отправляем код в облако

    git branch -M main
    git push -u origin main

После этого твой локальный код окажется на GitHub.
Теперь можно просто писать:

    git push

и все будущие коммиты будут лететь туда автоматически.

Шаг 5. Проверяем

Открой свой репозиторий на GitHub - если видишь файлы и коммит значит все работает.
Поздравляю, ты сделал первый пуш! Теперь твой код официально живет в облаке, а не только у тебя на диске.
